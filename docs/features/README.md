# Feature Documentation

This directory contains documentation for individual application features.

## Features

### [cold-areas/](./cold-areas/)

Cold area visualization and analysis - displaying and analyzing cold spots in the urban environment.

### [hsy-trees/](./hsy-trees/)

HSY tree data integration - importing, processing, and visualizing tree coverage data from Helsinki Region Environmental Services.

### [landcover-to-parks/](./landcover-to-parks/)

Land cover to parks conversion - processing land cover data and converting it to park representations.

### [socioeconomics/](./socioeconomics/)

Socioeconomic data visualization - charts and overlays for demographic and economic data.

## Feature Structure

Each feature directory typically contains:

- `README.md` - Feature overview, usage, and technical details
- Implementation notes and decisions
- Data sources and processing workflows
- Related components and services

## Adding New Features

When documenting a new feature:

1. Create a subdirectory with a lowercase-hyphenated name
2. Add a comprehensive README.md explaining:
   - Feature purpose and user value
   - Technical implementation
   - Data sources and dependencies
   - Related components and services
   - Usage examples or screenshots
